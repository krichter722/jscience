Provides support for exact or arbitrary precision measurements.

Measures as {@link javax.quantities quantities}:

The {@link org.jscience.physics.measures.Measure Measure} base class is parameterized with the quantity onto which the measure applies (Measure<Q extends Quantity>). If the quantity is not known then <?> can be used. For example:[code] Measure carMileage = Measure.valueOf(20, MILE.divide(GALLON_LIQUID_US)); Measure gazPrice = Measure.valueOf(1.2, EUR.divide(LITER)); // 1.2 €/L [/code] If the expected quantity result for a measure is known but has been lost due to the calculations performed; better than using <?>, the measure can be stated in a known unit for this quantity. It ensures dynamic check of the measure dimension/unit and avoid compiler warnings. For example:[code] // No problem here as the measure type is infered from the unit. Measure tripDistance = Measure.valueOf(400, KILO(SI.METER)); // Warning as the measure type is lost during calculation. Measure tripCost = tripDistance.divide(carMileage).times(gazPrice); // Better: Dimension check and no warning (USD is a Currency (Unit)) Measure tripCost = tripDistance.divide(carMileage).times(gazPrice).to(USD); [/code] It should be noted that this conversion is immediate if the specified unit is the actual unit for the measure (which is then returned unchanged).

Error calculations:

Measures take into account measurement and calculation errors. For example:[code] import static org.jscience.physics.units.SI.*; ... Measure x = Measure.valueOf(1.0, METER); Measure v = Measure.valueOf(0.01, METER_PER_SECOND); Measure t = Measure.valueOf(1.0, MICRO(SECOND)); for (int i = 0; i < 10000000; i++) { x = x.plus(v.times(t)); } MeasureFormat.setInstance(MeasureFormat.getExactDigitsInstance()); System.out.println(x); > 1.10000000 m The exact value is guaranteed to be in the range: ]1.09999999 m, 1.10000001 m[ The same calculation using primitive double type would have display: > 1.099999999392253 with no idea on the accuracy of the result.[/code]

Dimension checking.

The {@link org.jscience.physics.measures.Measure Measure} class is parameterized with the quantity onto which the measure applies (Measure<Q extends Quantity>). If the quantity is not known then <?> can be used. For example:[code] Measure carMileage = Measure.valueOf(20, MILE.divide(GALLON_LIQUID_US)); Measure gazPrice = Measure.valueOf(1.2, EUR.divide(LITER)); // 1.2 €/L [/code] The unit of a measure determinates its type. For example, Measure.valueOf("1 µm") and Measure.valueOf("1.2 ft") are Length measures (both "µm" and "ft" units are derived from SI.METER). Multiple physical models are supported (e.g. Standard, Relativistic, High-Energy, Quantum and Natural). The current model defines the conversions being allowed. For example:[code] Measure x = Length.valueOf(100, NonSI.INCH); LocalContext.enter(); try { RelativisticModel.select(); // Selects the relativistic model. x = x.plus(Measure.valueOf("2.3 µs")).to(METER); // Length and Duration can be added. System.out.println(x); Measure m = Quantity.valueOf("12 GeV").to(KILOGRAM); // Energy is compatible with mass (E=mc2) System.out.println(m); } finally { LocalContext.exit(); } > (692.06265340000008 ± 5.1E-13) m > (2.1391940763025056E-26 ± 4.3E-42) kg[/code]

Physical Constants

This package provides the latest physical {@link org.jscience.physics.measures.Constants constants} measurements with the best known accuracy (the more accurate the constant, the higher the precision of the calculations making use of these constants).
